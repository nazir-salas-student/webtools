My Experience in HTML

I have worked on HTML before in a different class.

Committing and pushing changes is a new one for me. I hope to address any gaps I have.

Some of the setup is a bit new to me but I hope to practice and get over the hurdle.

I do have working knowledge of Java, C++ and Python. I have used them to solvea variety of problems. One of my favorite projects was creating a program that had a graphical interface that looks at data from a SQL table and generates the information that the user selects for. This has quite a few applications. Some of them are projects that I am currently working for in my job.

I find that designing a web site and designing a program are very different. There are different rules to follow. If I have to describe it, HTML requires more artistic choices than other programming languages do. Which I find challenging but it is a quality that I have to figure out. And who knows, maybe I might make something that people will like.

But at the same time, there are a lot of common point. You have to solve a problem and design a way to remedy an issue that you encounter while programming. It is something that doesn't change between languages, and I expect that to be true for HTML as well.
